# 歷史演進與命名特例

本文件集中記錄考選部歷年國考資料中，容易在下載、分類、命名、入庫時踩坑的制度演進與官方命名特例。

原則：

- 官方 raw name 不覆寫，保留在 catalog、manifest、index 欄位。
- 檔名與資料夾可做必要標準化，例如全形括號轉半形括號。
- 制度交叉期的資料不刪、不合併成單一 PDF；以整理群組或 canonical mapping 歸併。
- 特殊考區、補辦考試、重新考試要保留獨立考試事件身分。

## 全形與半形括號

考選部官方 catalog 中，同一類科可能同時出現全形與半形括號，例如：

- `藥師（一）` / `藥師(一)`
- `藥師（二）` / `藥師(二)`
- `牙醫師（一）` / `牙醫師(一)`
- `牙醫師（二）` / `牙醫師(二)`

處理規則：

- catalog 與 manifest 保留官方 raw name。
- PDF 檔名與分類資料夾一律使用半形括號。
- 入庫時使用 `official_category_name` 保留來源名稱，另用 `normalized_category_name` / `group_name` 做歸併。

## 藥師：四年制到六年制交叉期

早期藥師考試為四年制畢業後考一次，後來改為六年制，考試制度變得接近醫師分階段考試。因此在 catalog 中會看到未分階段與分階段類科交叉出現。

目前已觀察到：

- `藥師`：民國 100-107 年
- `藥師（一）`：民國 103-105 年
- `藥師（二）`：民國 104-105 年
- `藥師(一)`：民國 105-115 年
- `藥師(二)`：民國 105-115 年

處理規則：

- 上述五種官方類科名稱都要完整收入。
- 不可因為 `藥師(一)/(二)` 已存在，就刪除早期 `藥師` 或全形括號過渡資料。
- PDF 檔名與資料夾使用半形括號。
- 索引與未來入庫時，整理群組 `group_name` 歸為 `藥師`。
- 若要做正式科目 canonical mapping，應另建 mapping 表，不覆寫官方 raw name。

## 中醫師：未分階段到分階段

中醫師早期也有未分階段與分階段並存的過渡期。

目前已觀察到：

- `中醫師`：民國 100-104 年
- `中醫師(一)`：民國 101-115 年
- `中醫師(二)`：民國 102-115 年

處理規則：

- `中醫師`、`中醫師(一)`、`中醫師(二)` 全部保留。
- 索引與未來入庫時，整理群組 `group_name` 歸為 `中醫師`。
- 不把未分階段資料硬改名為第一階段或第二階段。

## 中醫師 106111：花東考區補辦考試

民國 106 年中醫師資料中，`106110` 與 `106111` 會在同年同次序、同類科、同科目下造成檔名撞名。

已確認：

- `106110`：一般第 2 次考試，檔名使用 `1062_...`
- `106111`：花東考區補辦考試試題，檔名使用 `1063_...`

處理規則：

- `106111` 的 `exam_ordinal` 視為 `3`。
- PDF 放在 `106/第3次/`。
- 檔名使用 `1063_中醫師...`。
- PDF index notes 註記：`花東考區補辦考試試題`。
- 不再使用 `_E106110` 或 `_E106111` 表示這個已知案例。

## 醫事檢驗師科目名稱變化

醫事檢驗師在民國 102 年前後有科目名稱變化，可能代表考試大綱或命題範圍調整。

目前已觀察到：

- `臨床鏡檢學（包括寄生蟲學）`
  -> `醫學分子檢驗學與臨床鏡檢學（包括寄生蟲學）`
- `微生物學及臨床微生物學（包括細菌與黴菌）`
  -> `微生物學與臨床微生物學（包括細菌與黴菌）`

處理規則：

- PDF 檔名保留官方當年度科目名稱，只做括號半形化。
- 入庫時使用 `official_subject_name` 保存來源名稱。
- 另用 `canonical_subject_name` 或 subject mapping 表做跨年度歸併。

## 公職類科不要與專技類科混併

部分醫事相關類科同時存在公職版本，例如：

- `公職醫事檢驗師`
- `公職護理師`
- `公職營養師`
- `公職獸醫師`

處理規則：

- 目前 27 類整理範圍是專門職業及技術人員考試。
- `公職...` 類科不應與專技類科 silently merge。
- 若未來要收公職資料，應建立獨立整理群組或 scope。

- `scripts/download_moex_pdfs_from_catalog.py`
