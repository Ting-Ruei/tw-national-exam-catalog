# Contribution Guide

Contributions are welcome, especially:

- source catalog verification
- category and subject name normalization
- manual review of known official-page edge cases
- PDF asset manifests with SHA-256 hashes
- OCR / MinerU extraction quality reports
- structured question candidate schemas

## Principles

1. Preserve official raw values.
2. Add corrections as explicit derived fields or override tables.
3. Keep source URLs and stable registry keys.
4. Do not mix public-service exams into professional license scopes.
5. Do not publish AI-generated explanations as official answers.

## Suggested Issue Types

- `catalog-bug`: wrong or missing metadata
- `source-edge-case`: official page structure is unusual
- `normalization`: naming, parentheses, alias, or historical naming issue
- `asset-manifest`: PDF URL / hash / filename manifest contribution
- `parser-quality`: OCR or markdown extraction issue
- `question-structure`: structured JSON candidate issue
